To demonstrate how to set up Https in an embedded tomcat container using Spring Boot refer to the class TomcatSSLConfig.java



https://localhost:8443/PlayPen/



This code will create a tomcat coyote connector and configure accordingly, a self signed cert has been created and exists on the

classpath for use in the SSL handshake with the client browser.



As this is a self signed cert the browser will complain that the CA is not trusted this is fine in this case so accept the exception and the cert will be stored in the browser.



Doing the above through code will leave the http port (configured as 8080) active, if similar was to be done by Spring Boot configuration it would replace the

http connector with https connector.



{ The version of the spring boot starter pom.xml has also been upgraded to 1.2.3.RELEASE to take advantage some new annotations (mainly @SpringBootApplication.)}
